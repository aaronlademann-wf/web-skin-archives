Overview

Plugin dependencies

Some plugins and CSS components depend on other plugins.
If you include plugins individually, make sure to check for these dependencies in the docs.

All plugins depend on wf-vendor.js
Meaning that js/core/wf-js-vendor/dist/wf-vendor.js must be included before the plugin files.

Data attributes

You can use all Web Skin plugins purely through the markup API without writing a single line of JavaScript. This is Web Skin’s UI plugin API and should be your first consideration when using a plugin.

That said, in some situations it may be desirable to turn this functionality off. Therefore, we also provide the ability to disable the data attribute API by unbinding all events on the document namespaced with `'data-api'`. This looks like this:

$(document).off('.data-api');

Alternatively, to target a specific plugin, just include the plugin’s name as a namespace along with the data-api namespace like this:

$(document).off('.alert.data-api');

Duplicate component data attributes

Don’t mix data attributes from separate plugins on the same component. For example, a button cannot have data-toggle="tooltip" and data-toggle="modal". To accomplish this, use a separate wrapping element as shown below.

<div data-toggle="tooltip" title="Click me to launch a modal">
    <button type="button" data-toggle="modal">...</button>
</div>

Programmatic API

We also believe you should be able to use all Web Skin plugins purely through the JavaScript API. All public APIs are single, chainable methods, and return the collection acted upon.

$('.btn.danger').button('toggle')
                .addClass('downloading');

All methods should accept an optional options object, a string which targets a particular method, or nothing (which initiates a plugin with default behavior):

$('#myModal').modal();                        // initialized with defaults
$('#myModal').modal({ keyboard: false });     // initialized with no keyboard
$('#myModal').modal('show');                  // initializes and invokes show immediately

Each plugin also exposes its raw constructor on a Constructor property: $.fn.popover.Constructor. If you'd like to get a particular plugin instance, retrieve it directly from an element:

$('[rel="popover"]').data('wdesk.popover');

No conflict

Workiva developers will obviously use Web Skin plugins with other JS frameworks. In these circumstances, namespace collisions can occasionally occur. If this happens, you may call .noConflict on the plugin you wish to revert the value of.

var wdeskButton = $.fn.button.noConflict();   // return $.fn.button to previously assigned value
$.fn.button = wdeskButton;                    // give $().button the wdesk functionality

Events

Event namespacing

As of version 0.2.0 - all Web Skin plugin events are namespaced.

Web Skin provides custom events for most plugin’s unique actions. Generally, these come in an infinitive and past participle form - where the infinitive (ex. show) is triggered at the start of an event, and its past participle form (ex. shown) is trigger on the completion of an action.

All infinitive events provide preventDefault functionality. This provides the ability to stop the execution of an action before it starts.

$('#myModal').on('show.wdesk.modal', function (e) {
    if (! data) {
        return e.preventDefault(); // stops modal from being shown
    }
});

Transitions wdesk-transition.js

For simple transition effects, include wdesk-transition.js before all other Web Skin JS plugins.

About transitions

For simple transition effects, include transition.js once alongside the other JS files. If you’re using the compiled (or minified) bootstrap.js, there is no need to include this—it’s already there.

RequireJS usage

When using Web Skin plugins in a requireJS style dependency block, make sure that wdesk-transition.js is brought in as a dependency for all the other Web Skin plugins. Without it, the past participle form of plugin events won’t trigger correctly.

What’s inside

wdesk-transition.js is a basic helper for transitionEnd events as well as a CSS transition emulator. It’s used by the other plugins to check for CSS transition support and to catch hanging transitions.

Accessibility

The following behavior is built-in to the modal JS plugin in order to comply with WCAG 2.0:

  1. aria-hidden attribute is set to false when modals are opened, and true when closed
  2. Modals can be closed using the esc key on the keyboard
  3. When modals are opened, focus is given to the .modal element
  4. When modals are closed, focus is returned to the element that opened it
Related:  Modal Component HTML Markup Accessibility

Usage

Using hashbang targets with AngularJS

If you are using AngularJS for your Wdesk app, DO NOT USE <a> elements in combination with the data-toggle="modal" href="#toggleElem" attribute pattern. AngularJS reserves hashbang for routing - and in Internet Explorer, the whole page will refresh when you click an element that should only perform the toggle.

We recommend using another element like a <button> or <div>, while adding an appropriate role and tabindex attributes for accessibility.

<div role="button" tabindex="0" data-toggle="modal"
     data-target="#toggleElem"
     aria-controls="toggleElem">
    Toggle Me
</div>

See this article for more information.

Via HTML5 Data API

Activate a modal without writing JavaScript. Set data-toggle="modal" on a controller element, like a button, along with a data-target="#foo" or href="#foo" to target a specific modal to toggle.

<button type="button" data-toggle="modal" data-target="#myModal">Launch modal</button>

Via JavaScript API

Call a modal with id myModal with a single line of JavaScript:

$('#myModal').modal(options);

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-backdrop="".

Name type default description
backdrop boolean true Includes a modal-backdrop element. Alternatively, specify static for a backdrop which doesn’t close the modal on click.
backdropClass string '' Specify a custom css class for your modal backdrop
keyboard boolean true Closes the modal when escape key is pressed
show boolean true Shows the modal when initialized.
remote path false

If a remote url is provided, content will be loaded via jQuery’s load method and injected into the root .modal div. If you’re using the data api, you may alternatively use the href tag to specify the remote source. An example of this is shown below:

<a data-toggle="modal" href="remote.html" data-target="#modal">Click me</a>
container selector 'body' Define a DOM element on the page where modal content will appear. If this option is customized to not equal 'body', the modal plugin will produce a "contained" modal - like those used in the popover comments component. The overlaid CSS class is applied to this container element when the modal appears, and the modal element itself will receive a contained CSS class to help style it accordingly.
parentContainer selector 'body' This is always used in coordination with the container option above. Most of the time, this selector should match the container selector, but there are cases where you want it to be different. The modal-open CSS class is applied to this element when the modal appears.
sticky boolean false When set to true, a contained modal will not be dismissed when the user clicks outside of the .modal DOM element. Instead, use $('#myModal').modal('hide') to manually dismiss the modal.

Methods

Web Skin’s Modal class exposes a few methods for granular control via the JavaScript API.

Name description
.modal({options})

Activates your content as a modal. Accepts an optional options object.

$('#myModal').modal(); // no options
$('#myModal').modal({ keyboard: false });
.modal('toggle')

Manually toggles a modal.

$('#myModal').modal('toggle');
.modal('show')

Manually opens a modal.

$('#myModal').modal('show');
.modal('hide')

Manually hides a modal.

$('#myModal').modal('hide');

This event fires as soon as the modal-dialog element begins its show() transformation.

$('#myModal').on('show.wdesk.modal', function(event) { });
shown Past Participle

(will wait for css transitions to complete)

Accessibility

The following behavior is built-in to the dropdown JS plugin in order to comply with WCAG 2.0:

DOM Manipulation

  1. If no unique ID exists on a data-toggle="dropdown" element, one will be randomly generated and applied, and the menu that it controls will have a matching aria-labelledby attribute added to it when it is visible
    1. If a unique ID already exists, that will be used for the aria-labelledby reference
  2. The aria-expanded attribute value on data-toggle="dropdown" will be set to true when it’s dropdown menu is open, and false when it is closed
  3. All .hitarea elements within a visible dropdown menu will have their tabindex value changed to 0 to ensure keyboard navigation abilities on HTML elements that are not tabindexed by default

Keyboard Navigation

  1. When opening a dropdown menu, the first visible .hitarea element will gain focus
  2. When closing a dropdown menu, the triggering element (data-toggle="dropdown") will regain focus
  3. Dropdown menu items can be navigated via the up/down keys on a keyboard
  4. Dropdown menus can be opened via the spacebar key on a keyboard
  5. Open dropdown menus will close when pressing the tab, esc or spacebar key on a keyboard

For performance reasons, the Tooltip and Popover data-apis are opt in, meaning you must initialize them yourself via JS. If you would like to use their data-apis, just specify selector option when triggering via JavaScript.

Popovers in button groups and input groups require special setting

When using popovers on elements within a .btn-group or an .input-group, you'll have to specify the option container: 'body' (documented below) to avoid unwanted side effects (such as the element growing wider and/or losing its rounded corners when the popover is triggered).

Popovers on disabled elements require wrapper elements

To add a popover to a disabled or .disabled element, put the element inside of a <div> and apply the popover to that <div> instead.

Inserting content from existing DOM elements

Place a reference to an element’s CSS ID within the data-content attribute of the toggle button to inject the popover with that element’s content. Optionally, you can have the element you use for content removed after the popover is shown for the first time if you were only using it temporarily by adding data-content-container="temporary" to the toggle element.

Accessibility

The following behavior is built-in to the popover component JS plugin in order to comply with WCAG 2.0:

  1. A randomly-generated unique ID is added to the popover container
  2. The element that triggers the popover (data-toggle="popover") gets an aria-describedby attribute that matches the popover ID if the popover is visible.
  3. The aria-describedby attribute is removed from the triggering element when the popover is hidden.

Options

Options can be passed via data attributes or JavaScript. For data attributes, append any option name listed below to data-, as in data-container="body".

Name type default description
animation boolean true Apply a CSS fade transition to the popover
html boolean true Insert HTML into the popover. If false, jQuery’s text method will be used to insert content into the DOM. Use text if you’re worried about XSS attacks.
placement string | function 'bottom' How to position the popover - 'top' | 'bottom' | 'left' | 'right' | 'auto'

When 'auto' is specified, it will dynamically reorient the popover. For example, if placement is 'auto left', the popover will display to the left when possible, otherwise it will display right.

In practice, this is used to enable dynamic HTML content to have popovers added. See this informative JSFiddle example.
trigger string 'click' How a popover is triggered - 'click' | 'hover' | 'focus' | 'manual'

You may pass multiple triggers; simple separate them with a space.
title string | function '' Default title value if title attribute isn’t present
content string | function '' Default content value if data-content attribute isn’t present
delay number | object 0 Delay showing and hiding the popover (ms)
does not apply to manual trigger type

Number: Delay is applied to both hide and show methods

Object: delay: { show: 500, hide: 100 }
container string | false 'body' Appends the popover to a specific element container: '#myContainer'
modal boolean true When set to true the user can click anywhere outside the popover itself to close it (like a modal dialog).

Accessibility

The following behavior is built-in to the collapse JS plugin in order to comply with WCAG 2.0:

DOM Manipulation

  1. Will generate an aria-controls attribute based on the value of data-target or href attribute.
  2. Will control collapsibles via the aria-controls attribute with no-need for a redundant data-target or href attribute
    1. NOTE: This will likely become the default / recommended usage in a future release of Web Skin
  3. The aria-expanded attribute value on data-toggle="collapse" elements will be set to true when it’s target element is expanded, and false when it is collapsed
  4. The tabindex attribute value on data-toggle="collapse" elements will be changed to 0 when their target elements are expanded to ensure keyboard navigation abilities on HTML elements that are not tabindexed by default
  5. The aria-hidden attribute value on collapsible target elements will be set to false when it is expanded, and true when it is collapsed
  6. The tabindex attribute value on collapsible target elements will be changed to 0 when expanded and back to -1 when collapsed

Keyboard Navigation

  1. When expanding or collapsing a collapsible element, the corresponding data-toggle="collapse" element will keep focus to ensure proper keyboard navigation functionality
  2. data-toggle="collapse" elements can be navigated via the up/left and down/right keys on a keyboard
  3. Collapsible elements will expand/collapse when pressing the spacebar key on a keyboard

If you are using AngularJS for your Wdesk app, DO NOT USE <a> elements in combination with the data-toggle="collapse" href="#toggleElem" attribute pattern. AngularJS reserves hashbang for routing - and in Internet Explorer, the whole page will refresh when you click an element that should only perform the toggle.

We recommend using another element like a <button> or <div>, while adding an appropriate role and tabindex attributes for accessibility.

<div role="button" tabindex="0" data-toggle="collapse"
     data-target="#toggleElem"
     aria-controls="toggleElem">
    Toggle Me
</div>

See this article for more information.

Data API Compatibility

To initialize a collapsible element via JavaScript while maintaining the following behaviors of the data-attributes method:

  1. data- attribute options specified on the triggering element
  2. communication of state-based classes to the triggering element

You must refer to that triggering element (the one with [data-toggle=collapse] and [data-target]/[href]) during initialization in one of the following ways:

  1. an initial _relatedTarget passed into the .collapse() method
  2. an aria-labelledby attribute on the collapsible element

For more information, refer to the collapse constructor.

Placeholders for non-multiple selects

For <select multiple> elements, the data-placeholder attribute is all you’ll need to display a placeholder value when no selection has been made. However, for non-multiple select elements - you must add a selected option in order to display the placeholder. We suggest disabling it for accessibility purposes.

<select class="chosen form-control">
    <option disabled selected>Choose a Doctype</option>
    <option>10-K</option>
    <option>10-Q</option>
    ...
</select>

Guide your users through form completion by enforcing specific data formats.


    
Example input mask


Input masks can be used to force the user to enter data conform a specific format. Unlike validation, the user can’t enter any other key than the ones specified by the mask.

Affix wdesk-affix.js

    

    
Affix a UI component based on scroll position. The left navigation of the Web Skin docs is a live demo of the affix plugin.


    
Usage


Use the affix plugin via HTML5 Data API or manually with your own JavaScript.




    
Custom CSS Required

    
The affix plugin toggles between three classes, each representing a particular state: .affix, .affix-top, and .affix-bottom. You must provide the styles for these classes yourself (independent of this plugin) to handle the actual positions.


    
Here’s how the affix plugin works:


    
    
        
  1. To start, the plugin adds .affix-top to indicate the element is in its top-most position. At this point no CSS positioning is required.
    2. 
        
  2. Scrolling past the element you want affixed should trigger the actual affixing. This is where .affix replaces .affix-top and sets position: fixed; (provided by Bootstrap’s code CSS).
    3. 
        
  3. If a bottom offset is defined, scrolling past that should replace .affix with .affix-bottom. Since offsets are optional, setting one requires you to set the appropriate CSS. In this case, add position: absolute; when necessary. The plugin uses the data attribute or JavaScript option to determine where to position the element from there.
    4. 
    


    
Follow the above steps to set your CSS for either of the usage options below.






Via HTML5 Data API


To easily add affix behavior to any element, just add data-spy="affix" to the element you want to spy on. Use offsets to define when to toggle the pinning of an element.



<div data-spy="affix" data-offset-top="60" data-offset-bottom="200">...</div>




Via JavaScript API


Call the affix plugin via JavaScript:


$('#myAffix').affix({
    offset: {
        top: 100
      , bottom: function () {
            return (this.bottom = $('.body-footer').outerHeight(true));
        }
    }
});




Options


Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-offset-top="200".




    

        
            

                Name
                type
                default
                description
            

        
        
            

                offset
                number | function | object
                10
                
                    Pixels to offset from screen when calculating position of scroll. If a single number is provided, the offset will be applied in both top and bottom directions. To provide a unique, bottom and top offset just provide an object offset: { top: 10 } or offset: { top: 10, bottom: 5 }. Use a function when you need to dynamically calculate an offset.
