Dropdown menus

Toggleable, contextual menu for displaying lists of links.
Made interactive with the dropdown plugin.

Example

Wrap the dropdown's trigger and the dropdown menu within .dropdown, then add the menu's HTML.

  • Action
  • Another action
  • Something else here
  • Separated link
<div class="dropdown">
    <button id="dropdownMenu" class="btn dropdown-toggle">
        Dropdown
        <span class="caret"></span>
    </button>
    <ul role="menu" class="dropdown-menu" aria-labelledby="dropdownMenu">
        <li>
            <a class="hitarea" tabindex="-1" href="#">Action</a>
        </li>
        ...
        <li class="divider"></li>
        <li>
            <a class="hitarea" tabindex="-1" href="#">Separated link</a>
        </li>
        <li class="divider"></li>
        <li>
            <button class="hitarea" tabindex="-1" data-ng-click="doSomething()">Separated button</button>
        </li>
    </ul>
</div>

Clickable element class requirement

In order to provide maximum markup flexibility in Wdesk apps using Web Skin, all clickable elements within dropdowns require the inclusion of the .hitarea CSS class. This allows both <a> and <button> elements to be used interchangeably within dropdowns.

Aligning the menus

Add .pull-right to a .dropdown-menu or it's parent .dropdown to right align the dropdown menu.

<!-- on the menu -->
<ul class="dropdown-menu pull-right" role="menu" aria-labelledby="dropdownMenu">
    ...
</ul>

<!-- on the parent -->
<li class="dropdown pull-right">
    <!-- Link or button to toggle dropdown -->
    <ul class="dropdown-menu" role="menu" aria-labelledby="dropdownMenu">
        ...
    </ul>
</li>

Pager

Quick previous and next links for simple pagination implementations with light markup and styles. It's great for simple sites like blogs or magazines.

Default example

By default, the pager centers links.

  • Next

Aligned links

Alternatively, you can align each link to the sides:

  • ← Older
  • Newer →
<ul class="nav pager">
    <li class="previous"><a class="hitarea" href="#">&larr; Older</a></li>
    <li class="next"><a class="hitarea" href="#">Newer &rarr;</a></li>
</ul>

Clickable element class requirement

In order to provide maximum markup flexibility in Wdesk apps using Web Skin, all clickable elements within navbars require the inclusion of the .hitarea CSS class. This allows both <a> and <button> elements to be used interchangeably within navbars.

Block nav links

Simply add .nav-blocks to any .navbar-nav menu to get full-height "blocks" of links, perfect for housing icon-only hitareas. Primarily used in tablet viewer navbars.

        </button>
    </li>
</ul>

Centering icons

In order to center icons within .hitarea elements like those shown in the block nav link example above, add the .no-text helper class to zero-out any margin from the icon itself.

Disabled nav links

For any of the navbar nav links, add .disabled to the <li>, or attribute disabled to a button inside the nav's <li> to disable the link/button.

Link functionality of <a> not impacted

The .disabled class will only change the appearance of the link, not it's functionality. Either use custom JS to disable links, or use <button> elems with a disabled attribute instead (recommended).

Progress

Provide up-to-date feedback on the progress of a workflow or action with simple yet flexible progress components.

Cross-browser compatibility

Progress bars and spinners use CSS3 transitions and animations to achieve some of their effects. These features are not supported in MSIE 9 and below or older versions of Firefox. Web Skin uses animated .gif image fallbacks for the unsupported effects.

Progress bars

Basic

<div class="progress">
    <div class="progress-bar" style="width: 60%;"></div>
</div>

Additional colors

Progress bars use some of the same button and alert classes for consistent styles.

<div class="progress">
    <div class="progress-bar progress-bar-alt" style="width: 20%"></div>
</div>
<div class="progress">
    <div class="progress-bar progress-bar-success" style="width: 40%"></div>
</div>
<div class="progress">
    <div class="progress-bar progress-bar-warning" style="width: 60%"></div>
</div>
<div class="progress">
    <div class="progress-bar progress-bar-error" style="width: 80%"></div>
</div>

Animated / "Indeterminate"

Add .indeterminate to the progress elem to make the bar go 100% width to indicate an "indeterminate" amount of time remaining.

You can also use a .progress-label element to keep your users informed in the case there is an indeterminate state involved.

Processing… 
<div class="progress indeterminate">
    <div class="progress-bar"></div>
    <span class="progress-label">Processing&hellip;</span>
</div>

Stacked

Place multiple bars into the same .progress to stack them.

<div class="progress">
    <div class="progress-bar progress-bar-success" style="width: 35%"></div>
    <div class="progress-bar progress-bar-warning" style="width: 20%"></div>
    <div class="progress-bar progress-bar-error" style="width: 10%"></div>
</div>

Progress spinners

Basic

<i class="progress-spinner"></i>

Size variations

Progress spinners use built-in utility size classes to make it fit your layout just-right.

<i class="progress-spinner progress-spinner-huge"></i>
<i class="progress-spinner progress-spinner-gigantic"></i>
