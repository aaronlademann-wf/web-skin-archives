Overview

Plugin dependencies

Some plugins and CSS components depend on other plugins. If you include plugins individually, make sure to check for these dependencies in the docs. Also note that all plugins depend on jQuery (this means jQuery must be included before the plugin files).

Data attributes

You can use all Web Skin plugins purely through the markup API without writing a single line of JavaScript. This is Web Skin's UI plugin API and should be your first consideration when using a plugin.

That said, in some situations it may be desirable to turn this functionality off. Therefore, we also provide the ability to disable the data attribute API by unbinding all events on the body namespaced with `'data-api'`. This looks like this:

$(document).off('.data-api');

Alternatively, to target a specific plugin, just include the plugin's name as a namespace along with the data-api namespace like this:

$(document).off('.alert.data-api');

Programmatic API

We also believe you should be able to use all Web Skin plugins purely through the JavaScript API. All public APIs are single, chainable methods, and return the collection acted upon.

$('.btn.danger').button('toggle')
                .addClass('downloading');

All methods should accept an optional options object, a string which targets a particular method, or nothing (which initiates a plugin with default behavior):

$('#myModal').modal();                        // initialized with defaults
$('#myModal').modal({ keyboard: false });     // initialized with no keyboard
$('#myModal').modal('show');                  // initializes and invokes show immediately

Each plugin also exposes its raw constructor on a Constructor property: $.fn.popover.Constructor. If you'd like to get a particular plugin instance, retrieve it directly from an element:

$('[rel="popover"]').data('popover');

No conflict

WebFilings developers will obviously use Web Skin plugins with other JS frameworks. In these circumstances, namespace collisions can occasionally occur. If this happens, you may call .noConflict on the plugin you wish to revert the value of.

var wdeskButton = $.fn.button.noConflict();   // return $.fn.button to previously assigned value
$.fn.button = wdeskButton;                    // give $().button the wdesk functionality

Events

Event namespacing

As of version 0.2.0 - all Web Skin plugin events are namespaced.

Web Skin provides custom events for most plugin's unique actions. Generally, these come in an infinitive and past participle form - where the infinitive (ex. show) is triggered at the start of an event, and its past participle form (ex. shown) is trigger on the completion of an action.

All infinitive events provide preventDefault functionality. This provides the ability to stop the execution of an action before it starts.

$('#myModal').on('show.wdesk.modal', function (e) {
    if (! data) {
        return e.preventDefault(); // stops modal from being shown
    }
});

Transitions wdesk-transition.js

For simple transition effects, include wdesk-transition.js once alongside the other JS files.

About transitions

For simple transition effects, include transition.js once alongside the other JS files. If you're using the compiled (or minified) bootstrap.js, there is no need to include this—it's already there.

RequireJS usage

When using Web Skin plugins in a requireJS style dependency block, make sure that wdesk-transition.js is brought in as a dependency for all the other Web Skin plugins. Without it, the past participle form of plugin events won't trigger correctly.

What's inside

wdesk-transition.js is a basic helper for transitionEnd events as well as a CSS transition emulator. It's used by the other plugins to check for CSS transition support and to catch hanging transitions.

Usage

Via data attributes

Activate a modal without writing JavaScript. Set data-toggle="modal" on a controller element, like a button, along with a data-target="#foo" or href="#foo" to target a specific modal to toggle.

<button type="button" data-toggle="modal" data-target="#myModal">Launch modal</button>

Via JavaScript

Call a modal with id myModal with a single line of JavaScript:

$('#myModal').modal(options);

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-backdrop="".

Name type default description
backdrop boolean true Includes a modal-backdrop element. Alternatively, specify static for a backdrop which doesn't close the modal on click.
backdropClass string '' Specify a custom css class for your modal backdrop
keyboard boolean true Closes the modal when escape key is pressed
show boolean true Shows the modal when initialized.
remote path false

If a remote url is provided, content will be loaded via jQuery's load method and injected into the .modal-body. If you're using the data api, you may alternatively use the href tag to specify the remote source. An example of this is shown below:

<a data-toggle="modal" href="remote.html" data-target="#modal">Click me</a>
container selector 'body' Define a DOM element on the page where modal content will appear. If this option is customized to not equal 'body', the modal plugin will produce a "contained" modal - like those used in the popover comments component. The overlaid CSS class is applied to this container element when the modal appears, and the modal element itself will receive a contained CSS class to help style it accordingly.
parentContainer selector 'body' This is always used in coordination with the container option above. Most of the time, this selector should match the container selector, but there are cases where you want it to be different. The modal-open CSS class is applied to this element when the modal appears.
sticky boolean false When set to true, a contained modal will not be dismissed when the user clicks outside of the .modal DOM element. Instead, use $('#myModal').modal('hide') to manually dismiss the modal.

Methods

Name description
.modal({options})

Activates your content as a modal. Accepts an optional options object.

$('#myModal').modal(); // no options
$('#myModal').modal({ keyboard: false });
.modal('toggle')

Manually toggles a modal.

$('#myModal').modal('toggle');
.modal('show')

Manually opens a modal.

$('#myModal').modal('show');
.modal('hide')

Manually hides a modal.

$('#myModal').modal('hide');

Usage

Via data attributes

Add data-toggle="dropdown" to a link or button to toggle a dropdown.

<div class="dropdown">
    <a id="dLabel" class="dropdown-toggle hitarea" role="button"
       data-toggle="dropdown"
       href="#">
       Dropdown trigger <span class="caret"></span>
    </a>
    <ul class="dropdown-menu" role="menu" aria-labelledby="dLabel">
        <li role="presentation">
            <a class="hitarea" role="menuitem" tabindex="-1" href="#">...</a>
        </li>
        ...
    </ul>
</div>

To keep URLs intact, use the data-target attribute instead of href="#".

<div class="dropdown">
    <a id="dLabel2" class="dropdown-toggle hitarea" role="button"
       data-toggle="dropdown"
       data-target="#dd2"
       href="/page.html">
        Dropdown <span class="caret"></span>
    </a>
    <ul id="dd2" class="dropdown-menu" role="menu" aria-labelledby="dLabel2">
        <li role="presentation">
            <a class="hitarea" role="menuitem" tabindex="-1" href="#">...</a>
        </li>
        ...
    </ul>
</div>

Via JavaScript

Call the dropdowns via JavaScript:

$('.dropdown-toggle').dropdown();

Usage

Via data attributes

To easily add scrollspy behavior to your topbar navigation, just add data-spy="scroll" to the element you want to spy on (most typically this would be the body) and data-target=".navbar" to select which nav to use.

You'll want to use scrollspy with a .nav component of some kind.

<body data-spy="scroll" data-target=".navbar">
    ...
</body>

Via JavaScript

Call the scrollspy via JavaScript:

$('.navbar').scrollspy({ target: '#navbar-example' });

Resolvable ID targets required

Navbar links must have resolvable id targets. For example, a <a href="#home">home</a> must correspond to something in the DOM like <div id="home"></div>.

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-offset="".

Name type default description
offset number 10 Pixels to offset from top when calculating position of scroll.

Methods

Name description
.scrollspy('refresh')

When using scrollspy in conjunction with dynamic DOM manipulation or viewport resizing, you'll need to call the refresh method like so:

// add / remove DOM elements
$('[data-spy="scroll"]').each(function () {
    $(this).scrollspy('refresh');
});

// resize window
$(window).resize(function () {
    $('[data-spy="scroll"]').each(function () {
        $(this).scrollspy('refresh');
    });
});

Events

Event Description
activate

This event fires whenever a new item becomes activated by the scrollspy.

$('#myScrollspy').on('activate.wdesk.scrollspy', function () { });

Usage

Via data attributes

Activate a tab content pane without writing JavaScript. Set data-toggle="tab" on your tab menu hitarea element, along with a data-target="#foo" or href="#foo" to target a specific content pane to view.

<li><a class="hitarea" href="#profile" data-toggle="tab">Profile</a></li>

NOTE: To make tabs fade in, add .fade to each .tab-pane.

Via JavaScript

Enable tabbable tabs via JavaScript (each tab needs to be activated individually):

$('#myTab a').click(function (e) {
    e.preventDefault();
    $(this).tab('show');
});

You can activate individual tabs in several ways:

$('#myTab .hitarea[href="#profile"]').tab('show'); // Select tab by name
$('#myTab .hitarea:first').tab('show'); // Select first tab
$('#myTab .hitarea:last').tab('show'); // Select last tab
$('#myTab li:eq(2) .hitarea').tab('show'); // Select third tab (0-indexed)

    [
        'jquery',
        'angular',
        'wdesk-button'

                var $searchInput = $element.find('input[type="search"]');

                $scope.sampleQuery = '';
                $scope.updateQuery = function() {
                    // obviously you would wire this up to something real
                    searchForSomething($scope.sampleQuery);

                    // do something when we know that the user is actively searching for something...
                });

                $searchInput.on('clear', function(e) {
                    // do something when we know that the user has cleared the search input...
                });

                restrict: 'C', // bind to the search-box class

Chosen automatically sets the default field text by reading the select element's data-placeholder value. If no data-placeholder value is present, it will default to "Select an Option" or "Select Some Options" depending on whether the select is single or multiple.

<!-- Single Select -->
<select class="form-control" data-placeholder="Choose a Doctype">
    <option value="PDF">PDF</option>
    ...
</select>
<!-- Multiple Select -->
<select class="form-control" data-placeholder="Choose Doctypes" multiple>
    ...
</select>
<!-- Single Select w/ Groups -->
<select class="form-control" data-placeholder="Choose a Doctype">
    <optgroup label="Images">
        <option value="JPEG">JPEG</option>
        ...
    </optgroup>
    <optgroup label="Audio">
        ...
    </optgroup>
    <optgroup label="Video">
        ...
    </optgroup>
</select>
<!-- Multiple Select w/ Groups -->
<select class="form-control" data-placeholder="Choose Doctypes" multiple>
    ...
</select>

Usage

Via JavaScript

Enable manually with:

$('select.chosen').chosen({ options });

Options

Name type default description
allow_single_deselect boolean false When set to true on a single select, Chosen adds a UI element which selects the first elment (if it is blank).
disable_search boolean false When set to true, Chosen will not display the search field (single selects only).
disable_search_threshold integer 5 Hide the search input on single selects if there are fewer than (n) options.
enable_split_word_search boolean true By default, searching will match on any word within an option tag. Set this option to false if you want to only match on the entire text of an option tag.
inherit_select_classes boolean false When set to true, Chosen will grab any classes on the original select field and add them to Chosen's container div.
max_selected_options integer Infinity Limits how many options the user can select. When the limit is reached, the chosen:maxselected event is triggered.
no_results_text string 'No results match' The text to be displayed when no matching results are found. The current search is shown at the end of the text (e.g. No reults match "Bad Search").
placeholder_text_multiple string 'Select Some Options' The text to be displayed as a placeholder when no options are selected for a multiple select.
placeholder_text_single string 'Select an Option' The text to be displayed as a placeholder when no options are selected for a single select.
search_contains boolean false By default, Chosen's search matches starting at the beginning of a word. Setting this option to true allows matches starting from anywhere within a word. This is especially useful for options that include a lot of special characters or phrases in ()'s and []'s.
single_backstroke_delete boolean true By default, pressing delete/backspace on multiple selects will remove a selected choice. When false, pressing delete/backspace will highlight the last choice, and a second press deselects it.
width dimension inherit The width of the Chosen select container. By default, Chosen attempts to match the width of the select box you are replacing. If your select is hidden when Chosen is instantiated, you must specify a width or the select will show up with a width of 0.

In cases where your select is nested inside one of Web Skin's grid elements, the Chosen container will be set to a width of 100%.
display_disabled_options boolean true By default, Chosen includes disabled options in search results with styling to indicate it's disabled status. Setting this option to false will hide disabled results and exclude them from searches.
display_selected_options boolean true By default (In multiple selects only), Chosen includes selected options in search results similar to a "people picker" in a mail application. Setting this option to false will hide selected results and exclude them from searches.

UX NOTE: This is not recommended.

Methods

Name description
.chosen({options})

Activates your select elem as a chosen elem. Accepts an optional options object.

$('select.chosen').chosen(); // no options
$('select.chosen').chosen({ width: '95%' });

Events

Web Skin's chosen class exposes a few events for hooking into chosen's functionality.

Event Description
ready

Chosen has been fully instantiated (it also sends the chosen object as a parameter).

$('#myChosenElem').on('chosen:ready', function (e) { });
updated

This event can be fired when you want to dynamically update the options in a select element.

var $element = $('.chosen').chosen();
var datasource = attrs['options'];

scope.$watch(datasource, function() {
    // update the chosen options when the select elem has its options updated
    $element.trigger("chosen:updated");
});
change

Chosen triggers the standard DOM event whenever a selection is made (it also sends a selected or deselected parameter that tells you which option was changed).

$('#myChosenElem').on('change', function (e) { });
showing_dropdown

Triggered when Chosen's dropdown is opened (it also sends the chosen object as a parameter).

$('#myChosenElem').on('chosen:showing_dropdown', function (e) { });
hiding_dropdown

Triggered when Chosen's dropdown is closed (it also sends the chosen object as a parameter).

$('#myChosenElem').on('chosen:hiding_dropdown', function (e) { });
maxselected

Triggered if max_selected_options is set and that total is broken. (it also sends the chosen object as a parameter)

$('#myChosenElem').on('chosen:maxselected', function (e) { });
allselected

Triggered if the user opens the list of options after having already selected all available options.

$('#myChosenElem').on('chosen:allselected', function (e) { });

Affix wdesk-affix.js

The subnavigation on the left side of this documentation is a live demo of the affix plugin.

Usage

Via data attributes

To easily add affix behavior to any element, just add data-spy="affix" to the element you want to spy on. Then use offsets to define when to toggle the pinning of an element on and off.

<div data-spy="affix" data-offset-top="200">...</div>

Requires independent styling

Affix toggles between three states/classes: affix, affix-top, and affix-bottom. You must provide the styles for these classes yourself (independent of this plugin). The affix-top class should be in the regular flow of the document. The affix class should be fixed to the page. And affix-bottom should be positioned absolute. Note, affix-bottom is special in that the plugin will place the element with JS relative to the offset: { bottom: number } option you've provided.

Via JavaScript

Call the affix plugin via JavaScript:

$('#myAffix').affix({
    offset: {
        top: 100
      , bottom: function () {
            return (this.bottom = $('.page-footer').outerHeight(true));
        }
    }
});

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-offset-top="200".

Name type default description
offset number | function | object 10 Pixels to offset from screen when calculating position of scroll. If a single number is provided, the offset will be applied in both top and left directions. To listen for a single direction, or multiple unique offsets, just provide an object offset: { x: 10 }. Use a function when you need to dynamically provide an offset (useful for some responsive designs).
